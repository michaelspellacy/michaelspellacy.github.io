Keyboard Event Not Firing On Elements Containing ::before Pseudo-element When NVDA Running

Issue:
When NVDA (Non-Visual Desktop Access) is running, keyboard events do not appear to fire on interactive elements that include a ::before pseudo-element that may contain text content, emoticons, or CSS counters and that also include a display property, like display: block;. An event will fire on pointer click, but not when the Enter key is clicked. On the button element, the Space key should also fire an event, but does not. When either the display or content properties are removed, the event will then fire as it should. A null content property, like content: "", used with a display property, does not appear to trigger the issue. The ::after pseudo-element is also uneffected here.
Platform(s):
Chrome: Version 106.0.5249.119 (Official Build) (64-bit)
Edge: Version 106.0.1370.47 (Official build) (64-bit)
NVDA: 2022.3.1

Steps to Duplicate Issue

  1. Install and open NVDA
  2. On this page:
    • Open DevTools and access "Console"
    • Tab through page and click on each interactive element below
    • The function should write the word "Success" to the console on every click, which is not happening on elements that contain a ::before. The event works as it should when elements that contain an ::after pseudo-element are clicked.
